19 abstract asset IAMObject extends Information
20 user info: "An IAM object represents the base logic shared by all assets used for Identity and Access Management roles(Identity, Group, Privileges)."
21 {
22 # notPresent [Disabled]
23 user info: "This defense is used to reason about the attack vectors introduced by this asset. It should be used to model the possibility that the IAM object does not actually exist."
24 developer info: "Enabling this defense should make the asset behave as if it did not exist, as such all of its impactful attack steps should be disabled by it."
25 -> successfulAssume,
26 successfulLockout
27
28 | attemptAssume @hidden
29 developer info: "Intermediate attack step to allow for defenses."
30 -> successfulAssume
31
32 & successfulAssume @hidden
33 developer info: "Intermediate attack step to model defenses."
34 -> assume
35
36 | assume
37 user info: "When an attacker is able to assume an IAM asset the privileges associated with it can always be exploited, other behaviour is asset specific."
38 -> execPrivApps.authenticate,
39 highPrivApps.authenticate,
40 lowPrivApps.specificAccessAuthenticate,
41 readPrivData.authorizedReadFromIAM,
42 writePrivData.authorizedWriteFromIAM,
43 writePrivData.authorizedApplicationRespondConnectFromIAM,
44 deletePrivData.authorizedDeleteFromIAM,
45 managedIAMs.attemptAssume,
46 subprivileges.attemptAssume
47
48 | deny
49 user info: "Denying Identity, Group, or Privileges means that they are unavailable to legitimate users and which may be locked out as a result."
50 +> attemptLockout
51
52 | write
53 user info: "Overwriting Identity, Group, or Privileges means that the attacker is able to assume them."
54 +> attemptAssume
55
56 | attemptLockout @hidden
57 developer info: "Intermediate attack step to allow for defenses."
58 -> successfulLockout
59
60 & successfulLockout @hidden
61 developer info: "Intermediate attack step to model defenses."
62 -> lockout
63
64 | lockout {A}
65 user info: "The Identity, Group, or Privileges has been locked out and cannot be used by legitimate users. This means that the applications that they can execute and data they can read might also be denied if all of the access roles that could execute or read them, respectively, have been locked out."
66 -> execPrivApps.denyFromLockout,
67 readPrivData.denyFromLockout,
68 subprivileges.lockout

71 asset Identity extends IAMObject
72 user info: "An identity models an IAM identity that should then be associated with privileges on other instances."
73 developer info: "An identity can be visualised as a group of assumable roles that can be associated with many credentials."
74 {
75 # notPresent [Disabled]
76 user info: "Prevent unsafe user activity too if the Identity is assumed to not exist."
77 +> successfulUnsafeUserActivity
78
79 | assume {C,I,A}
80 user info: "After authentication or compromise of an account/identity, assume its privileges."
81 developer info: "This is both legitimate and illegitimate access! Also assume all the privileges of the parent identities (on the above level/inherited by this identity) because those represent the group of (inherited) roles."
82 +> parentId.attemptAssume,
83 memberOf.attemptAssume
84
85 & lockoutFromCredentials @hidden
86 developer info: "Only lockout an identity if all of the Credentials that could be used to authenticate have been denied."

88
89 | lockout

91 +> memberOf.lockoutFromMembers
92
93 !E missingUser @hidden
94 developer info: "If there are no Users asociated with this Identity we make the worst case scenario assumption regarding the strength of the Credentials belonging to it."
95 <- users
96 -> credentials.weakCredentials
97
98 | attemptUnsafeUserActivity @hidden
99 developer info: "Intermediate attack step to allow for defenses."
100 -> successfulUnsafeUserActivity
101
102 & successfulUnsafeUserActivity @hidden
103 developer info: "Intermediate attack step to model defenses."
104 -> unsafeUserActivity

114 asset Privileges extends IAMObject
115 user info: "A privileges asset can be associated with another IAMObject to reason about a specific subset of given privileges on Applications and Data."
116 modeler info: "The privileges asset is not meant to be used normally when assigning permissions. Identities and Groups should be directly associated with the relevant assets. The purpose of the Privileges asset is to be used as a speculative asset when reasoning about additional privileges that a particular Group or Identity may have as a result of either poor configurations or the attacker's actions."

118 # notPresent [Disabled]
119 user info: "Prevent unsafe user activity too if the Privileges are assumed to not exist."

128 developer info: "Intermediate attack step to allow for defenses."

143 asset Group extends IAMObject
144 user info: "A group is a way to group together identities and/or other groups. This allows the expression of hierarchical IAM (structured inheritance)."
145 modeler info: "Groups can be used instead of nested identities to make the model more intuitive and clearer to understand."
146 {
147 | assume {C,I,A}
148 user info: "If an identity of a group is compromised then the whole group(i.e. all other privileges of the group) should be considered as compromised. Furthermore, the parent groups should also be considered compromised."
149 developer info: "The parent groups should be compromised because all the privileges of the parent groups are inherited on the children groups but lower children groups should not be compromised because lower levels might have inherited plus additional privileges."
150 +> parentGroup.attemptAssume
151
152 & lockoutFromMembers @hidden
153 developer info: "If all of the Identities belonging to a Group are lockedout we assume that the Group as a whole is lockedout."
154 -> attemptLockout
155 }

157 asset Credentials extends Information
158 user info: "Credentials can be used to get access to an Identity, but they can also be used as an encryption/signing key for Data."
159 modeler info: "Credentials represent a variety of access control mechanism(e.g. username and password pair, keycards, biometric data)."
160 {
161 # notDisclosed [Enabled]
162 user info: "Describes the case where the password/credential is leaked to some location, it can then be available to the attacker."
163 -> useLeakedCredentials
164
165 # notGuessable [Enabled]
166 user info: "Describes if the credentials can be guessed by the attacker(i.e. they represent a password or passphrase)."
167 -> guessCredentials
168
169 # unique [Enabled]
170 user info: "Describes if the credentials are known to be unique and therefore cannot be used in a credentials reuse attack."
171 -> credentialsReuse,
172 propagateOneCredentialCompromised
173
174 # notPhishable [Disabled]
175 user info: "Describes if the credentials cannot be phished(they are biometric, e.g. fingerprints or iris scans, or physical, e.g. access cards or keys, elements)."
176 -> credentialTheft
177
178 | read
179 user info: "If the attacker is able to read the information containing credentials we assume that they are compromised."
180 +> attemptUse
181
182 | write
183 user info: "If the attacker is able to write the information containing credentials we assume that they have changed them for the authentication process."
184 +> attemptUse
185
186 | deny
187 user info: "If the attacker is able to deny the information containing credentials we assume that they have denied them for the authentication process."
188 +> identities.lockoutFromCredentials
189
190 | useLeakedCredentials [EasyAndCertain]
191 user info: "If the password/credential is leaked to some location, it can then be available to the attacker and therefore it can be used."
192 -> attemptUse
193
194 | attemptCredentialsReuse @hidden
195 developer info: "Intermediate attack step to allow for defenses."
196 -> credentialsReuse,
197 requiredFactors.attemptCredentialsReuse
198
199 & credentialsReuse @hidden
200 developer info: "The user is reusing credentials which means that the attacker is able to compromise all of the non-unique credentials associated with this user."
201 -> attemptUse
202
203 | attemptUse @hidden
204 developer info: "In order to use these credentials the attacker may require additional factors."
205 -> use
206
207 & use {C}
208 user info: "The attacker is using the credentials to perform a legitimate authentication."
209 -> identities.attemptAssume,
210 encryptedData.accessDecryptedData,
211 signedData.accessSpoofedData,
212 attemptPropagateOneCredentialCompromised,
213 credentials.use, // these credentials may serve as additional required factors for other credentials in a multi-factor authentication
214 hashes.attemptUse, // the hashes generated based on these credentials are also compromised as the attacker is assumed to know which hashing functions were used
215 origCreds.guessCredentialsFromHash
216
217 | attemptPropagateOneCredentialCompromised @hidden
218 developer info: "Intermediate attack step to allow for defenses."
219 -> propagateOneCredentialCompromised
220
221 & propagateOneCredentialCompromised @hidden
222 developer info: "Intermediate step used to propagate the credentials reuse attack step."
223 -> credentials*.identities.users.oneCredentialCompromised
224
225 | attemptCredentialTheft @hidden
226 developer info: "The attacker is attempting to access the credentials."
227 -> credentialTheft,
228 requiredFactors.attemptCredentialTheft
229
230 & credentialTheft @hidden
231 developer info: "The attacker is able to steal the credentials."
232 -> attemptUse
233
234 !E missingIdentity @hidden
235 developer info: "If there are no Identities asociated with these Credentials we make the worst case scenario assumption regarding the strength of the Credentials."
236 <- identities
237 -> weakCredentials
238
239 | weakCredentials @hidden
240 developer info: "Intermediate step used to represent how weak the credentials the user employs are. This is inversely related to the securityAwareness defense on the User asset associated with the Identity to which these Credentials belong. If either the User or Identity associations are missing the assumption is made that the crentials are weak and therefore guessable by the attacker."
241 -> guessCredentials,
242 requiredFactors.weakCredentials
243
244 !E credsHashed @hidden
245 developer info: "If there are no associated hashes the assumption is that the credentials are guessable without any additional requirements."
246 <- hashes
247 -> guessCredentialsFromHash
248
249 | guessCredentialsFromHash @hidden
250 developer info: "If hashes are associated with this Credentials asset the assumption is that one of those hashes is required by the attacker to be able to perform a guessing attack."
251 -> guessCredentials
252
253 & guessCredentials @hidden [HardAndUncertain]
254 developer info: "The attacker can try to just guess a set of credentials. The likelihood of succeeding is dependent on how strong the credentials are."
255 modeler info: "The guessability of the Credentials is influenced by the notGuessable defense on this asset and the securityAwareness defense on the User associated with the Identity that these Credentials belong to. If either the User or Identity associations are missing the assumption is made that the crentials are guessable and only the notGuessable defense would play a role in restricting this attack step."
256 -> attemptUse
257 }

260associations {
261 Credentials [encryptCreds] 0..1 <-- EncryptionCredentials --> * [encryptedData] Data
262 user info: "Encrypted data can be associated with the relevant encryption credentials."
263 Credentials [signingCreds] 0..1 <-- SigningCredentials --> * [signedData] Data
264 user info: "Signed data can be associated with the relevant signing credentials."
265 Credentials [hashes] * <-- HashedCredentials --> 0..1 [origCreds] Credentials
266 user info: "Credentials can be associated with their hashed counterparts. Multiple hashes can be generated for the same Credentials, but a hash is unique to one set of Credentials."
267 Credentials [credentials] * <-- ConditionalAuthentication --> * [requiredFactors] Credentials
268 user info: "Credentials can be associated with other Credentials to depict conditional authentication procedures, such as multi-factor authentication."
269 // ### Access Control happens below
270 Identity [identities] * <-- IdentityCredentials --> * [credentials] Credentials
271 Identity [parentId] * <-- CanAssume --> * [childId] Identity
272 user info: "Starting from a parent Identity, the child Identities can be assumed due to inheritance."
273 Group [memberOf] * <-- MemberOf --> * [groupIds] Identity
274 Group [parentGroup] * <-- MemberOf --> * [childGroups] Group
275 // Then, Access Control on application level
276 IAMObject [executionPrivIAMs] * <-- ExecutionPrivilegeAccess --> * [execPrivApps] Application
277 user info: "Every application executes on a system with privileges of a specified identity on the system. If the application is compromised then the privileges should be compromised."
278 IAMObject [highPrivAppIAMs] * <-- HighPrivilegeApplicationAccess --> * [highPrivApps] Application
279 user info: "High privilege application access on an Application results in the (full) access/compromise of the application and all the child applications."
280 IAMObject [lowPrivAppIAMs] * <-- LowPrivilegeApplicationAccess --> * [lowPrivApps] Application
281 user info: "Low privilege application access on an Application allows only the local interaction with the application and all the specified privileges."
282 // Finally, Access control on data
283 IAMObject [readingIAMs] * <-- ReadPrivileges --> * [readPrivData] Data
284 IAMObject [writingIAMs] * <-- WritePrivileges --> * [writePrivData] Data
285 IAMObject [deletingIAMs] * <-- DeletePrivileges --> * [deletePrivData] Data
286 // Associations for the Privileges asset
287 IAMObject [IAMOwners] * <-- HasPrivileges --> * [subprivileges] Privileges
288 user info: "Identities, Groups, and Privileges may have a subset of Privileges that we logically separate to model specific scenarios, such as partial lockouts or speculating about possible privileges."
289 // Self-referential associations for the IAMObject asset
290 IAMObject [managers] * <-- AccountManagement --> * [managedIAMs] IAMObject
291 user info: "Identities, Groups, and Privileges may have account management roles for other Identities, Groups, and Privileges."
