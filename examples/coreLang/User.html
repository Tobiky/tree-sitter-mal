17category User {
18
19 asset User
20 user info: "This asset represents the user themselves. It is the suitable attack surface for social engineering attacks."
21 {
22 # noPasswordReuse [Enabled]
23 user info: "If one credential of that user is compromised there is a probability that all other credentials of that user are also compromised."
24 -> passwordReuseCompromise
25
26 # noRemovableMediaUsage [Enabled]
27 user info: "The user does not utilise removable media and therefore the attack steps associated with it cannot be reached."
28 -> successfulDeliverMaliciousRemovableMedia
29
30 # securityAwareness
31 user info: "The security awareness of the user makes it less likely that social engineering would be successful and reduces the likelihood that the user will engage in unsafe behaviour."
32 -> securityAwarenessBypassed,
33 weakCredentials
34
35 | bypassSecurityAwareness [VeryHardAndUncertain]
36 user info: "Security awareness can be bypassed."
37 -> securityAwarenessBypassed
38
39 | securityAwarenessBypassed @hidden
40 developer info: "Security awareness is bypassed either because it was not set, the attacker was able to circumvent it through additional effort, or we simply assume that everyone has an off day every now and then."
41 -> successfulSocialEngineering,
42 successfulDeliverMaliciousRemovableMedia
43
44 | oneCredentialCompromised @hidden
45 developer info: "This intermediate attack step is needed in order to block passwordReuseCompromise when no other credential is first compromised."
46 -> passwordReuseCompromise
47
48 & passwordReuseCompromise @hidden
49 developer info: "If one reused credential of that user is compromised then, all other credentials of that user can also be compromised."
50 -> userIds.credentials.attemptCredentialsReuse
51
52 | weakCredentials @hidden
53 developer info: "Intermediate attack step that defines the strength of all of the Credentials associated with the user. This is inversely proportional with the user's securityAwareness defense."
54 -> userIds.credentials.weakCredentials
55
56 | attemptSocialEngineering @entrypoint
57 user info: "An attacker can try to perform social engineering techniques such as phishing and inducing the user to perform malicious actions."
58 developer info: "Intermediate attack step to allow for defenses and trigger bypasses."
59 modeler info: "Social engineering is modeled by having the attacker trigger this attack step on the target Users."
60 -> successfulSocialEngineering,
61 bypassSecurityAwareness
62
63 & successfulSocialEngineering @hidden
64 developer info: "Intermediate attack step to model defenses."
65 -> socialEngineering
66
67 | socialEngineering @hidden
68 developer info: "Currently social engineering entails phishing credentials and tricking the user into unsafe actions that expose Applications to the attacker. In the future, other social engineering techniques should be added to this attack step."
69 -> credentialTheft,
70 unsafeUserActivity
71
72 | attemptDeliverMaliciousRemovableMedia @entrypoint
73 user info: "An attacker can try to deliver a removable media drive(i.e. USB drive) containing malicious code to a location accessible to the target users."
74 developer info: "Intermediate attack step to allow for defenses and trigger bypasses."
75 modeler info: "Social engineering is modeled by having the attacker trigger this attack step on hthe target Users."
76 -> successfulDeliverMaliciousRemovableMedia,
77 bypassSecurityAwareness
78
79 & successfulDeliverMaliciousRemovableMedia @hidden
80 developer info: "Intermediate attack step to model defenses."
81 -> deliverMaliciousRemovableMedia
82
83 | deliverMaliciousRemovableMedia @hidden [Exponential(0.01)]
84 developer info: "The user is assumed to connect the removable media containing the malicious code to all of the hardware systems that they have access to. Since this is expected to unwittingly happen as part of their normal operations the attack exposes their regular security context."
85 -> userIds.attemptAssume,
86 physicalZones.hardwareSystems.attemptUnsafeUserActivity,
87 hardwareSystems.attemptUnsafeUserActivity
88
89 | credentialTheft @hidden [HardAndUncertain]
90 developer info: "Trick the user into providing their credentials."
91 modeler info: "Distribution: Bernoulli(0.5) * Exponential(0.1), source: Sommestad (2011) Password authentication attacks: a survey of attacks and when they will succeed, suggest to use Bernoulli(0.05)"
92 -> userIds.credentials.attemptCredentialTheft
93
94 | unsafeUserActivity @hidden [Exponential(0.03)]
95 developer info: "The user can engage in unsafe behaviour that could allow the attacker to gain access to the applications the user has access to."
96 modeler info: "If the attacker is not able to reach (or be reached) via any of the Applications this garnered access does not materialise into any actual attack vectors."
97 -> userIds.attemptAssume,
98 userIds.attemptUnsafeUserActivity
99 }
100}
