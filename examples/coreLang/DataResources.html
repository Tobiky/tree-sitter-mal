20 user info: "Represents any type of information that might be contained inside Data. This include general purpose information, but also serves as the basis for more specialized types, such as Credentials and SoftwareProduct. Information represents the abstract concept, while Data represent an actual instance of information stored or transmitted digitally."

275 Data [containingData] * <-- DataContainment --> * [containedData] Data

278 user info: "An application should be able to contain some data."

280 user info: "An application can also send some data that are supposed to be transferred over a network or connection. This expresses an active connection."
281 Data [receivedData] * <-- ReceiveData --> * [receiverApp] Application
282 user info: "An application can also receive some data that are supposed to be transferred over a network or connection. This expresses an active connection."
283 Data [transitData] * <-- DataInTransit --> * [transitNetwork] Network
284 user info: "A network can also contain some data that are supposed to be network-wide available."
285 Data [hostedData] * <-- DataHosting --> 0..1 [hardware] Hardware

287 Data [containerData] * <-- InfoContainment --> * [information] Information
288 user info: "Data can contain information, as for example credentials."
289 Data [dataReplicas] * <-- Replica --> * [replicatedInformation] Information
290 user info: "Information can be replicated across multiple data assets that offer redundancy."
291 // ### Application dependence associations
292 /* Dependence is used to represent various situations where the operations
293 * of the Application are impacted by modifying/denying the Data/Information
294 * it depends upon. Examples of this type of behavior are configurations,
295 * plugins, libraries, or other input data that is critical to the purpose
296 * of the software component represented by the Application. This
297 * association should only be used for circumstances where the workflow of
298 * the Application is not altered, but the Data/Information are modified
299 * within the regular operating process. For situations where abnormal
300 * behaviour is induced to impact the Application SoftwareVulnerabilities
301 * should be used instead.
302 */
303 Data [dataDependedUpon] * <-- Dependence --> * [dependentApps] Application
304 user info: "Data can be specified as a dependence for an Application."
305 Information [infoDependedUpon] * <-- Dependence --> * [dependentApps] Application
306 user info: "Information can be specified as a dependence for an Application."
