308 user info: "An application can communicate / be exposed on a network."
309 modeler info: "This is DEPRECATED and should only be used by fully aware modelers."
310 developer info: "This can equivalently be modeled using a ConnectionRule but this direct approach should be prefered in the case where the application is “located” in the same network as the host on which it is running."
311 Application [applications] * <-- ApplicationConnection --> * [appConnections] ConnectionRule
312 user info: "An application can communicate over one or more (mesh type) bidirectional connections."
313 modeler info: "ConnectionRules associated with Applications are practically expressing a traffic flow."
314 Application [inApplications] * <-- InApplicationConnection --> * [ingoingAppConnections] ConnectionRule
315 user info: "Two or more applications can also be inter-connected via a uni-directional connection. In means that this is the server side application, awaiting incoming connections."
316 Application [outApplications] * <-- OutApplicationConnection --> * [outgoingAppConnections] ConnectionRule
317 user info: "Two or more applications can also be inter-connected via a uni-directional connection. Out means that this is the client side application initiating connections."
318 Network [networks] * <-- NetworkConnection --> * [netConnections] ConnectionRule
319 user info: "Two or more networks can also be inter-connected via a bidirectional connection."
320 Network [inNetworks] * <-- InNetworkConnection --> * [ingoingNetConnections] ConnectionRule
321 user info: "Two or more networks can also be inter-connected via a uni-directional connection. In means that ingoing traffic to the network is allowed."
322 Network [outNetworks] * <-- OutNetworkConnection --> * [outgoingNetConnections] ConnectionRule
323 user info: "Two or more networks can also be inter-connected via a uni-directional connection. Out means that outgoing traffic from the network is allowed."
324 Network [diodeInNetworks] * <-- DiodeInNetworkConnection --> * [diodeIngoingNetConnections] ConnectionRule
325 user info: "Two or more networks can also be inter-connected via a uni-directional connection that acts like a diode. That means that no responses can come back after connection initiation."
